# 에러 코드 명세서

`ErrorType` enum 기준 전체 에러 코드 카탈로그. `GlobalExceptionHandler` 가 `BusinessException` 의 `ErrorType` 을 읽어 HTTP 응답 본문에 `message` + `errorCode` 형태로 직렬화한다.

## 공통 응답 포맷

```json
{
  "message": "올바른 입력값 형식이 아닙니다.",
  "errorCode": "COMMON400_004"
}
```

| 필드 | 타입 | 설명 |
| --- | --- | --- |
| `message` | `String` | 사용자/개발자용 한국어 메시지 |
| `errorCode` | `String` | 프론트 분기·로그 분석용 식별자 |

---

## 코드 네이밍 규칙

```
{도메인}{HTTP_상태}_{일련번호}
```

- 도메인 prefix: `COMMON`, `RESERVATION`, `RESERVATION_TIME`, `RESERVATION_WAIT`, `THEME`, `AUTH`
- HTTP 상태: 400, 401, 403, 404, 409, 422, 500
- 일련번호: 같은 (도메인, 상태) 조합 내 순번 (001부터)

예) `RESERVATION_WAIT409_001` = 예약 대기 도메인의 409 Conflict 1번 에러.

## 1. 공통 (COMMON)

| 에러 코드 | HTTP | 메시지 | 발생 상황 |
| --- | --- | --- | --- |
| `COMMON400_001` | `400` | 유효하지 않은 요청필드입니다. | `@Valid` 검증 실패 (`MethodArgumentNotValidException`) |
| `COMMON400_002` | `400` | 경로 변수(PathVariable)가 누락됐습니다. | URL path 변수 누락 |
| `COMMON400_003` | `400` | 쿼리 스트링이 누락됐습니다. | 필수 쿼리 파라미터 누락 |
| `COMMON400_004` | `400` | 올바른 입력값 형식이 아닙니다. | JSON 파싱 실패 / Content-Type 불일치 |
| `COMMON400_005` | `400` | 올바른 쿼리 스트링 형식이 아닙니다. | 쿼리 파라미터 타입 변환 실패 |
| `COMMON400_006` | `400` | 유효하지 않은 쿼리 스트링 값입니다. | `@Validated` + `ConstraintViolationException` |
| `COMMON500_001` | `500` | 예기치 못한 예외가 발생했습니다. | 핸들되지 않은 모든 예외 (fallback) |

## 401 vs 403 구분

- `401 Unauthorized` = **인증 실패** (당신이 누구인지 모름)
- `403 Forbidden` = **인가 실패** (누구인지는 알지만 권한 없음)

따라서 비로그인 사용자가 매니저 API 를 호출하면 `401` (AUTH401_002) 가 먼저 발생하고, 로그인은 했지만 USER 인 경우에 `403` (AUTH403_001) 가 발생한다.
