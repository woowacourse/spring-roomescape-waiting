# 학습 로그 #38

**학습 범위**: 메타 — **ACL(부패방지계층)은 목적의 수단이지 법이 아니며, 그 "선"은 권위 있는 레퍼런스/스펙으로 확인한다.** (같은 세션의 *설계 적용 인출*은 [[37_결제연동-설계인출]].)

분류: 외부 API 연동 / 결제 — 메타·원칙 (재방문: #33 책임의 위치, #31 Saga)

## 1. 계기 — 세션의 메타 절반
오늘 **코드 적용**(엔드포인트·주문·검증순서·포트&어댑터의 *설계 인출*)은 [[37_결제연동-설계인출]]에 있다. 이 로그는 그 §5(포트&어댑터 경계)에서 갈라져 나온 **"예외를 어디까지 도메인으로 번역하나 / ACL은 얼마나 엄격해야 하나"** 논쟁과 그 메타-교훈을 다룬다.

## 3. 오늘의 알맹이 — ACL은 "법"이 아니라 "수단"
- **ACL = Anti-Corruption Layer (부패 방지 계층)**: 외부(토스)와 내 도메인 *사이*의 **번역+격리 막** — 외부의 개념·변경이 *도메인 안으로* 새어 들어와 오염시키는 걸 막는다(통역사 겸 문지기). ⚠️ Access Control List(접근제어 목록) 아님. 공교롭게 오늘 IDOR 수정에서 *진짜* 접근제어(소유권 검증)도 다뤄 두 ACL이 한 세션에 동시 등장.
- **체득 격차(중요)**: 사이클 마무리에서 (1) ACL을 Access Control List로 잘못 풀고, (2) "정확히 정의 못 하겠다"고 했다. 즉 **쓸 줄은 알아도(적용·코치 반박·레퍼런스 검증까지 함) 한 문장 정의는 못 하는 상태** — 절차지(procedural) > 선언지(declarative) 격차. 방향도 "외부로 퍼짐"으로 거꾸로 잡았다(정답: 외부의 것이 *안으로* 못 퍼지게).
- 코치(AI)가 "예외 이름에도 Toss 없어야 한다"며 ACL을 **레퍼런스보다 엄격하게** 적용 → 리뷰 #4가 "PaymentController가 TossProperties를 안다 = 누수"로 잡음.
- 내가 제동: *"그냥 TossExceptionHandler 만들면 되잖아. ACL이 꼭 지켜야 하는 법이야? 결국 변경 범위 줄이려는 거잖아."*
- **레퍼런스(woowacourse/spring-third-party-api, learning-test-1 `complete`) 직접 확인** → 어댑터가 `TossPaymentException`(client 패키지, Toss-named, code 보유)을 던지고, 웹 컨트롤러가 그걸 `import`·`catch`해서 `getCode()`를 읽는다. = 내가 처음 제안한 바로 그 패턴.
- **레퍼런스가 그은 선**: 토스 DTO/JSON 파싱은 어댑터에 가둠 + `PaymentService`는 토스 DTO를 모름. *단, 타입 예외(TossPaymentException)는 경계를 넘어도 OK.* (코치 잣대보다 느슨, 실용 쪽.)
- **교훈**: 원칙은 가이드다. *어디에 선을 긋는지*는 권위 있는 레퍼런스/스펙 + 목적(변경범위·테스트가능성)으로 정한다. 코치도 과적용했고, 내가 레퍼런스로 교정했다. → 최종 코드를 레퍼런스 패턴으로 리팩터링(`TossPaymentException` + `@ExceptionHandler`).
- **마지막 구조(내가 원한 것)**: 토스 예외 핸들러를 전역 대신 **전용 `@RestControllerAdvice`(payment.toss, `@Order(HIGHEST_PRECEDENCE)`)** 로 분리 → `GlobalExceptionHandler`는 토스 무지, web↔토스 결합은 advice 한 곳에 **국소화**. *결합을 없앤 게 아니라 가장 덜 퍼지는 곳에 가둔 것.* (순서 함정: TossPaymentException은 RuntimeException이라 글로벌 `handleUnexpected`가 선점하지 않게 우선순위를 올려야 함 — 테스트로 확인.)

## 4. "확인"의 세 종류 (자기정리에서 흐릿했던 지점)
- **이 설계가 맞는 선택인가?** → 레퍼런스/스펙/목적으로 *판단*. (오늘 한 것)
- **경계가 실제로 지켜지나?** → ArchUnit 같은 *아키텍처 테스트*로 강제. 리뷰도 "그런 테스트 없음" 지적. (오늘 안 함 → i+1)
- **동작이 맞나?** → 단위/E2E 테스트(228 그린). 설계 옳고그름과 별개.

## 5. 리뷰가 확인해준 것 — 파킹한 arc는 진짜였다
