## 2. 이번 타임의 학습 전략

### 학습 과정

#### 세션(쿠키) 방식

1. 로그인 성공 → 서버가 세션에 memberId 저장
2. 응답에 `Set-Cookie: JSESSIONID=...` 포함
3. 이후 요청마다 브라우저가 자동으로 `Cookie: JSESSIONID=...` 전송
4. 서버가 JSESSIONID로 세션 조회 → memberId 확인

---

#### 토큰(헤더) 방식

1. 로그인 성공 → 서버가 토큰 생성 후 응답에 포함
2. 클라이언트가 토큰 저장 (로컬 스토리지 등)
3. 이후 요청마다 클라이언트가 직접 `Authorization: Bearer ...` 헤더 추가
4. 서버가 토큰 검증

---

#### 핵심 비교

| | 세션 방식 | 토큰 방식 |
|---|---|---|
| 저장 주체 | 서버 | 클라이언트 |
| 즉시 로그아웃 | 서버에서 세션 삭제 → 가능 | 클라이언트가 들고 있어서 어려움 |
| 탈취 시 대응 | 서버에서 세션 삭제 가능 | 만료 전까지 막기 어려움 |
| 서버 확장 | 세션 공유 문제 (Redis로 해결 가능) | 무상태 → 확장 유리 |
| 모바일 앱 | 쿠키 자동 관리 없어 불편 | 모든 HTTP 클라이언트에서 동일 |

---

#### 결론

은총알은 없다. 상황에 맞게 선택:
- 브라우저 기반 + 즉시 무효화 중요 → 세션
- 모바일 / 다양한 클라이언트 / 서버 확장 중요 → 토큰
