# 사이클 1 기능 목록

### 1단계

- [x] 사용자는 예약된 슬롯(날짜+시간+테마)에 대한 대기를 신청할 수 있다.
    - [x] 신청한 순번대로 대기 순번이 정해진다.
    - [x] 하나의 슬롯에는 사용자당 하나의 대기 신청만 가능하다.
- [x] 사용자는 본인의 대기를 취소할 수 있다.

### 2단계

- [x] 사용자는 예약 목록과 함께 대기 목록을 조회할 수 있다.
    - [x] 예약 및 대기 상태를 구분할 수 있다.
    - [x] 대기 상태면 대기 순번도 함께 조회한다.

# 사이클 2 기능 목록

### 1단계

- [x] 예약 취소 시 대기 1번이 자동으로 예약으로 전환된다.
- [x] 다음 경우에는 대기 순번이 재정렬된다.
    - [x] 대기 1번이 예약으로 승격된 경우
    - [x] 예약이 취소된 경우

<details>
<summary>자동 승격 선택 이유</summary>

- 예약이 취소되었을 경우 대기를 걸어둔 사용자가 자동으로 예약으로 승격되는 것이 관리자의 일을 줄인다고 판단

- 관리자가 수동으로 대기를 승격시킬 경우 취소된 예약의 승격을 잊거나,
  순번을 잘못 파악하여 2번 대기자를 승격시키는 등 실수가 발생할 수 있고,
  혹은 관리자가 대기를 승격시키려는 순간 대기자가 대기를 취소하는 상황이 발생할 수 있다고 판단

</details>
