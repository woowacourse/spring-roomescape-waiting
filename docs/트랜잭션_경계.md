# Transaction Boundary

- 시나리오 흐름 별 묶어야 하는 작업, 묶지 않아도 되는 작업 분리

### 1️⃣ 예약이 취소될 때 대기 순번의 1번이 예약으로 바뀐다. (변경도 마찬가지)

```text
 예약 취소
    ⬇️
 예약 대기 슬롯 확인    ➡️ (대기 ❌) 트랜잭션 Commit
    ⬇️ (대기 🅾️)
 대기 순번의 1번을 예약 추가
    ⬇️
 대기 순번 1번 삭제
    ⬇️
 트랜잭션 Commit
```

- 예약 취소와 대기 1번의 예약 전환은 하나의 트랜잭션으로 묶는다.
- 대기 1번을 예약으로 전환하는 과정에서 실패하면 예약 취소도 롤백한다.
- 예약 취소만 반영된 중간 상태가 노출되면, 해당 슬롯이 비어 있는 것처럼 보일 수 있고 요구사항의 의도와 달라진다.
