# 05 · 4단계 — 예약 대기 관리 (자동 승인 / 트랜잭션 경계)

> **설계 판단의 벽.** 어드민 대기 관리 + 예약 취소 시 자동 승인. 이 단계가 만드는 고유 관찰 = **flush 순서**.
> ⚠️ **내 코드 특수 상황**: `delete-before-insert` 승급 흐름 + `@Transactional`이 **이미 JDBC로 구현됨**(Cycle 2
> ReservationPromotionTransactionTest 등). 그래서 이 단계는 그 흐름을 JPA로 다시 표현하고 **flush 순서를 관찰**하는 게 핵심.
> 커밋: `[4단계] ...` / 모델: Sonnet. **도달한 만큼을 본다** — 수동까지만 하고 자동을 미완으로 둬도 한 선택.
