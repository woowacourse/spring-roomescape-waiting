# Spring Roomescape Waiting

방탈출 테마 예약과 예약 대기를 관리하는 Spring Boot 애플리케이션입니다. 
사용자는 날짜, 테마, 시간을 선택해 예약하거나 이미 예약된 시간에 대기를 신청할 수 있고, 관리자는 테마와 예약 시간을 관리할 수 있습니다.

## 기술 스택

- Java 21
- Spring Boot 3.4.4
- Spring Web, Thymeleaf, JDBC, Validation
- H2 in-memory database
- Gradle
- JUnit 5, RestAssured, Mockito

## 주요 기능

### 사용자 기능

- 전체 테마 조회
- 최근 7일 예약 수 기준 인기 테마 TOP 10 조회
- 날짜, 테마, 시간 선택 후 예약 생성
- 이미 예약된 시간에 예약 대기 생성
- 예약자 이름으로 내 예약과 예약 대기 조회
- 본인 이름 확인 후 예약 수정, 예약 삭제, 예약 대기 삭제

### 관리자 기능

- 예약 시간 등록, 조회, 삭제
- 테마 등록, 조회, 삭제
- 예약이 연결된 시간 또는 테마 삭제 방지

## 테스트

```bash
./gradlew test
```

테스트는 도메인 단위 테스트, Repository 테스트, Controller 통합 테스트를 포함합니다. 
테스트 환경에서는 `src/test/resources/application.properties`에서 `spring.sql.init.data-locations=`를 비워 초기 데이터 로딩을 제어합니다.

## 비즈니스 규칙

- 지난 날짜 또는 이미 지난 시간에는 예약/예약 대기를 생성할 수 없습니다.
- 같은 날짜, 시간, 테마 조합에는 하나의 확정 예약만 존재할 수 있습니다.
- 예약 대기는 이미 확정 예약이 존재하는 날짜, 시간, 테마에만 생성할 수 있습니다.
- 확정 예약자와 같은 이름으로 같은 예약에 대기를 생성할 수 없습니다.
- 예약 또는 예약 대기 삭제 시 요청자의 이름이 기존 예약자 이름과 일치해야 합니다.
- 예약 수정 시 기존 예약자 이름이 일치해야 하며, 수정하려는 시간은 비어 있어야 합니다.
- 예약이 존재하는 예약 시간 또는 테마는 삭제할 수 없습니다.
- 예약 대기 순번은 같은 날짜, 시간, 테마 안에서 대기 ID 순서로 계산됩니다.
- 예약이 삭제된다면, 모든 대기자의 순번이 하나 씩 당겨지고, 1번 대기자는 자동으로 예약 상태로 전환될 예정입니다.(미구현)
