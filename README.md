## 시나리오

- 예약자 1명이 예약을 한다.
- 예약 정보에 맞는 스케줄을 생성한다 (Date, Time, Theme으로 이루어진 스케줄 생성)
- 해당 스케줄에 예약이 존재하는지 확인한다.
- 예약이 존재하지 않는 경우 예약 생성
- 예약이 존재하는 경우 대기 생성

- 예약자 1명이 예약을 삭제한다.
- 예약 삭제 시 해당 스케쥴에 대기 인원이 없다면 예약과 스케쥴 모두 삭제한다.
- 예약 삭제 시 대기 인원이 있다면 Waiting 정보로 예약을 변경한다. (Reservation 에 있는 MemberId를 Waiting에 있는 Member로 변경하고 Waiting 정보 삭제한다.)
