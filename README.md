## 1. 기능 요구 사항
### 1. 예약
1. 예약자와 관리자는 예약을 생성한다.
2. 모든 예약을 조회한다. 
3. 특정(본인의) 예약을 삭제한다.
4. 특정(본인의) 예약을 변경한다.
5. 특정(본인의) 모든 예약을 조회한다.
    - 유저 목록이 없기 때문에 해당 이름이 DB에 진짜 있는 지는 검증하지 않는다.

### 2. 타임대
1. 타임대를 추가한다.
2. 전체 타임과 예약 가능 상태를 조회한다.
3. 특정 타임대를 삭제한다.

### 3. 테마
1. 테마를 생성한다.
2. 특정 테마를 삭제한다.
3. 테마들을 조회한다.(옵션: 날짜)
4. 1주 인기 테마들을 조회한다

### 4. 스케줄
1. 스케줄을 생성한다.
2. 특정 스케줄을 삭제한다.
3. 모든 스케줄을 조회한다.
4. 특정 스케줄을 삭제한다.

### 5. 예외 상황
1. 스케줄 테이블에 등록되지 않은 스케줄(날짜, 시간, 테마)로 예약을 시도할 시 예외가 발생한다.
2. 예약이나 스케줄에 등록되어있는 시간을 삭제하려할 시 예외가 발생한다.
3. 예약이나 스케줄에 등록되어있는 테마를 삭제하려할 시 예외가 발생한다.
4. 스케줄이 같은 예약을 중복예약할 수 없다.
5. 현재 날짜보다 이전 날짜의 스케줄로 예약 추가, 삭제, 수정을 할 수 없다.
6. 특정 값으로 조회시 데이터가 없다면 예외가 발생한다.

### 6. 인증/인가
1. 토큰 기반 로그인/로그아웃을 제공한다.
2. 로그인하지 않은 사용자는 `/login`, `/logout`, `/themes/popular` 외 요청 시 인증 예외가 발생한다.
3. 관리자 전용 자원(`times`, `themes` 일부, `schedules`)에 일반 사용자가 접근하면 인가 예외가 발생한다.

### 7. 예약 대기 신청/취소
1. 이미 다른 사용자에 의해 예약된 스케줄(날짜+시간+테마)에 대기를 신청할 수 있다.
2. 같은 스케줄에 대한 대기는 신청 순서대로 순번이 부여된다.
3. 같은 사용자가 같은 스케줄에 중복 대기할 수 없다.
4. 사용자는 본인의 대기를 취소할 수 있다.
5. 대기 신청 api를 추가한다. -> 대기 버튼을 통해
6. 대기 취소는 soft delete말고 hard delete를 사용한다. -> 내 예약정보에서 취소 버튼을 통해

### 8. 내 예약 목록 조회 (상태 구분)
1. 사용자의 예약과 대기가 상태로 구분되어 함께 표시된다.
2. 대기에는 본인의 대기 순번도 함께 보여준다.

### 9. 예약 승격
1. 기존 예약을 취소할 시 첫 대기자는 예약으로 자동 승격된다.
2. 기존 예약을 변경할 시 첫 대기자는 예약으로 자동 승격된다.

## 2. 성공/실패 응답 포맷

### 성공 응답 포맷 예시
```json
{
   "success": "boolean",
   "data": "object | array | null",
   "error": null
}
```

### 실패 응답 포맷 예시
```json
{
  "success": "boolean",
  "data": null,
  "error": {
    "code": "string",
    "message": "string"
  }
}
```

## 3. API 명세서
[API 명세서](https://www.notion.so/API-4c86f304080b82a8b9d581262209fcbe?source=copy_link) 

주요 HTTP 상태 코드:
- `200 OK`: 조회/수정/로그인 성공
- `201 Created`: 생성 성공
- `204 No Content`: 삭제/로그아웃 성공
- `400 Bad Request`: 요청 값 검증 실패
- `401 Unauthorized`: 로그인 실패, 미인증 요청
- `403 Forbidden`: 관리자 권한 없음
- `404 Not Found`: 리소스를 찾을 수 없음
- `409 Conflict`: 중복 예약/중복 리소스/사용 중 리소스 충돌
- `500 Internal Server Error`: 서버 내부 오류
