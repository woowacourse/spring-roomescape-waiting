# 예약 시간 인수 테스트 시나리오

예약 시간 등록, 중복 등록 방지, 예약 가능 시간 조회 흐름을 검증한다.

## 시나리오 목록

| 시나리오 | 검증 내용 |
| --- | --- |
| 예약 시간 목록 조회 | 생성한 예약 시간이 목록에 포함되는지 검증 |
| 중복 예약 시간 생성 | 같은 예약 시간을 다시 생성할 수 없는지 검증 |
| 예약 가능 시간 조회 | 특정 날짜와 테마 기준으로 예약 가능 여부가 내려오는지 검증 |

## Gherkin

```gherkin
Feature: 예약 시간 관리 기능

  Scenario: 예약 시간 목록 조회
    Given 예약 시간 생성을 요청하고
    When 예약 시간 목록 조회를 요청하면
    Then 생성한 예약 시간이 포함된 예약 시간 목록을 응답받는다.

  Scenario: 중복 예약 시간 생성
    Given 예약 시간 생성을 요청하고
    When 같은 예약 시간 생성을 다시 요청하면
    Then 중복 예약 시간 생성은 실패한다.

  Scenario: 예약 가능 시간 조회
    Given 테마 생성을 요청하고
    Given 예약 시간 생성을 요청하고
    Given 새로운 예약 시간 생성을 요청하고
    Given 특정 날짜와 테마에 예약 생성을 요청하고
    When 특정 날짜와 테마의 예약 가능 시간 조회를 요청하면
    Then 예약된 시간은 예약 불가로 응답받는다.
    Then 예약되지 않은 시간은 예약 가능으로 응답받는다.
```
