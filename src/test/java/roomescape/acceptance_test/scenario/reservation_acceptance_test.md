# 예약 인수 테스트 시나리오

예약 생성, 조회, 수정, 취소와 예약 대기 정책 예외 흐름을 검증한다.

## 시나리오 목록

| 시나리오 | 검증 내용 |
| --- | --- |
| 관리자 예약 목록 조회 | 생성한 예약이 관리자 목록에 포함되는지 검증 |
| 관리자 예약 취소 | 관리자가 예약을 취소할 수 있는지 검증 |
| 내 예약 목록 조회 | 사용자 이름 기준으로 본인 예약을 조회하는지 검증 |
| 예약 수정 | 예약 날짜와 시간을 변경하고 테마는 유지되는지 검증 |
| 이미 시작된 예약 수정 실패 | 이미 시작된 예약을 수정할 수 없는지 검증 |
| 지난 날짜와 시간으로 예약 수정 실패 | 과거 날짜와 시간으로 예약을 수정할 수 없는지 검증 |
| 다른 사용자의 예약 수정 실패 | 다른 사용자의 예약을 수정할 수 없는지 검증 |
| 내 예약 취소 | 본인 예약을 취소하고 취소 상태로 조회되는지 검증 |
| 빈 시간대 예약 확정 | 빈 시간대에 예약하면 확정 예약이 되는지 검증 |
| 예약 대기 생성 | 이미 확정 예약이 있는 시간대에 예약하면 대기 예약이 되는지 검증 |
| 예약 대기 순번 | 같은 시간대의 대기 예약에 생성 순서대로 대기 순번이 부여되는지 검증 |
| 대기 예약 확정 전환 | 확정 예약을 취소하면 첫 번째 대기 예약이 확정되는지 검증 |

## Gherkin

```gherkin
Feature: 예약 관리 기능

  Scenario: 관리자 예약 목록 조회
    Given 테마 생성을 요청하고
    Given 예약 시간 생성을 요청하고
    Given 예약 생성을 요청하고
    When 관리자 예약 목록 조회를 요청하면
    Then 생성한 예약이 포함된 관리자 예약 목록을 응답받는다.

  Scenario: 관리자 예약 취소
    Given 예약 생성을 요청하고
    When 생성한 예약 취소를 요청하면
    Then 생성한 예약 취소가 성공한다.

  Scenario: 내 예약 목록 조회
    Given 특정 사용자 이름으로 예약 생성을 요청하고
    When 내 예약 목록 조회를 요청하면
    Then 특정 사용자의 예약이 포함된 예약 목록을 응답받는다.

  Scenario: 예약 수정
    Given 테마 생성을 요청하고
    Given 예약 시간 생성을 요청하고
    Given 변경할 예약 시간 생성을 요청하고
    Given 예약 생성을 요청하고
    When 예약 날짜와 시간 수정을 요청하면
    Then 예약 수정이 성공한다.
    Then 예약 날짜와 시간은 요청한 값으로 응답받는다.
    Then 예약 테마는 기존 테마로 응답받는다.

  Scenario: 이미 시작된 예약 수정 실패
    Given 예약 생성을 요청하고
    Given 현재 시간이 예약 시작 이후가 되고
    When 예약 날짜와 시간 수정을 요청하면
    Then 예약 수정 실패 응답을 받는다.

  Scenario: 지난 날짜와 시간으로 예약 수정 실패
    Given 예약 생성을 요청하고
    Given 현재 시간이 변경하려는 예약 날짜와 시간 이후가 되고
    When 지난 날짜와 시간으로 예약 수정을 요청하면
    Then 예약 수정 실패 응답을 받는다.

  Scenario: 다른 사용자의 예약 수정 실패
    Given 특정 사용자 이름으로 예약 생성을 요청하고
    Given 다른 사용자 이름으로 새로운 예약 생성을 요청하고
    When 다른 사용자의 이름으로 예약 수정을 요청하면
    Then 예약 수정 실패 응답을 받는다.

  Scenario: 내 예약 취소
    Given 특정 사용자 이름으로 예약 생성을 요청하고
    When 내 예약 취소를 요청하면
    Then 내 예약 취소가 성공한다.
    Then 내 예약 목록 조회 시 취소한 예약은 취소 상태로 응답받는다.

  Scenario: 빈 시간대 예약은 확정된다
    Given 예약할 날짜, 시간, 테마가 있고
    When 빈 시간대에 예약 생성을 요청하면
    Then 예약 상태는 CONFIRMED로 응답받는다.

  Scenario: 이미 확정 예약이 있는 시간대의 예약은 대기가 된다
    Given 같은 날짜, 시간, 테마에 확정 예약이 있고
    When 같은 시간대에 다른 사용자가 예약 생성을 요청하면
    Then 예약 상태는 WAITING으로 응답받는다.
    Then 대기 순번은 1번으로 응답받는다.
    Then 내 예약 목록 조회 시 대기 순번은 1번으로 응답받는다.

  Scenario: 같은 시간대의 대기 예약은 생성 순서대로 대기 순번이 부여된다
    Given 같은 날짜, 시간, 테마에 확정 예약이 있고
    When 첫 번째 대기 예약 생성을 요청하고
    When 두 번째 대기 예약 생성을 요청하면
    Then 첫 번째 대기 예약의 대기 순번은 1번이다.
    Then 두 번째 대기 예약의 대기 순번은 2번이다.

  Scenario: 확정 예약을 취소하면 첫 번째 대기 예약이 확정된다
    Given 같은 날짜, 시간, 테마에 확정 예약이 있고
    Given 같은 시간대에 대기 예약이 있고
    When 확정 예약 취소를 요청하면
    Then 예약 취소가 성공한다.
    Then 대기 예약은 CONFIRMED 상태로 응답받는다.
```
